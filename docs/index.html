GUI-RC: identify consensus regions across sampling for precise grounding.   ·   GUI-RCPO: convert region consistency into rewards for test-time RL on unlabeled data.

News

  • 2025-08-09 — We release our codes.
  • 2025-08-07 — We release our paper: Test-Time Reinforcement Learning for GUI Grounding via Region Consistency.

Motivation

Current GUI grounding approaches rely heavily on large-scale pixel-level annotations and training-time optimization, which are expensive, inflexible, and difficult to scale to new domains. We observe that when GUI models generate multiple predictions, spatial overlaps across generations naturally reflect the model's localization confidence. This leads to a critical question:

Can we leverage test-time computation to enhance GUI grounding performance without additional labeled data?

We introduce GUI-RC and GUI-RCPO to unlock the untapped potential of region consistency, enabling models to self-improve without labeled data.

Highlights

  • GUI-RC (Region Consistency Voting): spatial voting over multiple samples to find consensus regions — +2–3% gains without training.
  • GUI-RCPO (Region Consistency Policy Optimization): transform region consistency into self-supervised rewards for test-time RL — additional +4–5% gains.
  • Self-Bootstrapping: applying GUI-RC after GUI-RCPO yields even higher accuracy — progressive self-improvement.
  • Robustness: consistent improvements across models and benchmarks.

Quick Start

Setup

conda create -n ttrl4gui python=3.10
conda activate ttrl4gui
bash setup.sh

Running GUI-RC

cd TTRL4GUI/GUI-RC
python evaluation.py
Edit in evaluation.py:
  • MODEL_PATH: model path
  • QUESTION_TEMPLATE: prompt template (default: Qwen2.5‑VL)
  • TEMPERATURE: sampling temperature (default: 0.7)
  • SAMPLE_NUM: number of samples (default: 64)
  • POINT_EXPAND_SIZE: click region expansion size (default: 50)

Training GUI-RCPO

cd TTRL4GUI/VLM-R1
sh run_scripts/run_gui_rcpo_Qwen2.5-VL-3B.sh

data_paths="${PROJECT_ROOT}/data/your_dataset/your_data.jsonl"
image_folders="${PROJECT_ROOT}/data/your_dataset/images"
model_path="your_model_path"

Training data example: TTRL4GUI/data/screenspot/example_training_data.jsonl
